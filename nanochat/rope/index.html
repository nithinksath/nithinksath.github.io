Embeddings & attention

RoPE vs. learned positional embeddings

BPE tells the model what token it sees. Position encoding tells it where the token is. Learned embeddings memorize one vector per position; RoPE uses a predictable rotation inside attention.

1. Keep the concepts separate

BPEtext → token IDs
Token embeddingID → meaning vector
Positionwhere it occurs
tokens: ["The", "cat", "sat"]
token IDs: [464, 3797, 3332]
positions: [0, 1, 2]

The position number is usually an index or an input to a formula—not a one-dimensional feature simply appended to the token.

2. Learned positional embeddings are a lookup table

xᵢ = token_embedding(tokenᵢ) + position_embedding(i)

Position 101 selects row 101 from a learned table. The row is a full vector.

Rows 100 and 101 are learned independently. Nothing in the representation itself says they are neighbors.

3. The drawbacks, precisely

No neighbor rule

Position 101 is not generated from position 100 plus one step.

Distance is indirect

The model must infer that positions 101 and 106 are five apart from arbitrary vectors.

Hard table boundary

If training ends at position 2047, row 2048 is missing or untrained.

4. RoPE turns position into rotation

qᵢ′ = R(i)qᵢ    kⱼ′ = R(j)kⱼ

Query and key dimensions are paired into 2D planes. Each pair rotates by an angle determined by the token position.

5. Why relative distance appears naturally

(R(i)q)ᵀ(R(j)k) = qᵀR(j − i)k

Each vector uses an absolute position, but their dot product depends on j − i, the relative offset.

6. Why different rotation speeds?

Clock analogy: different dimension pairs rotate like second, minute, and hour hands. Fast pairs distinguish nearby positions; slow pairs preserve information over longer distances.

7. Side-by-side summary

QuestionLearned position embeddingRoPE
RepresentationOne learned vector per positionDeterministic rotation
Applied where?Added to token representationUsually queries and keys
Relative distance built in?NoYes, through j − i
Learned position parameters?YesNo table
Beyond training length?Not naturallyRotation can be computed, though quality is not guaranteed
One-sentence intuition: learned positional embeddings give every position an arbitrary ID card; RoPE gives every step a predictable turn, so attention can sense how far apart two tokens are.
